## 一、核心认知转变：这张图改变了一切

### 触发事件

Boyang 上传了一张产品实际输出图：一张公园照片，上面叠加了多个白色手写风格的印尼语标注，每个标注指向照片中的具体元素，配有诗意的口语描述、星星装饰和云朵文字框。

### 认知前后对比

**之前的理解**（文档描述的产品）：
- 用户拍照 → BLIP-2 理解场景 → GLM 生成印尼语故事文本 → 展示文字 → Anki 导出
- 输出是：一段故事文字 + 填空题

**看到图之后的理解**（真实产品形态）：
- 用户拍照 → 生成带有印尼语标注的美图 → 可直接分享到小红书/朋友圈 → Anki 导出
- 输出是：一张视觉上完整的、可分享的语言图片日记

### 关键洞察

这张图的性质不是"学习材料"，而是**一种新的内容形式**：用目标语言做图片日记。

产品的实际输出比所有文档描述的要性感得多。现有文档里大量篇幅在讲技术架构，但产品最大的差异化价值是这张图本身的视觉形态——这在所有对外沟通里都应该排在第一位。

图中印尼语示例（可供参考）：
- `pohon rindang — adem banget, kayak dipeluk pelan`（茂密的树——很凉快，像被轻轻拥抱）
- `langit biru — cerahnya bikin hati ikut ringan`（蓝天——明亮得让心情也变轻）
- `gazebo putih — tempat duduknya tenang, pengen lama-lama di sini`（白色凉亭——坐在这里很平静，想待很久）
- `hari ini rasanya sederhana tapi hangat banget di hati`（今天感觉很简单，但心里很温暖）

这种表达是地道的印尼语口语，不是教科书语言，正好符合 i+1 可理解输入的要求。

## 二、产品形态重新定义

### 修订后的一句话定义

> 拍下你眼前的世界，Lensa 用印尼语把它标注出来，变成你可以分享、可以复习的语言图片日记。
>
> Point. Shoot. Learn. Share.

### 双输出结构（修订版）

| 输出 | 内容 | 用途 | 优先级 |
|---|---|---|---|
| 标注美图 | 照片 + 印尼语标注文字 + 视觉装饰 | 即时分享、社交传播 | Priority 1 |
| Anki .apkg | 图中新词 + 发音 + 例句 | 长期记忆、持续复习 | Priority 2 |

这两个输出服务两种不同需求，互不冲突，互相强化：
- 美图让分享变自然，是产品的营销引擎
- Anki 文件让学习持久，是产品的留存引擎

### 内容即营销

用户分享出去的标注美图本身就是产品广告。每张图右下角加小 logo + "Lensa · 学印尼语"标签，自然品牌曝光，零推广成本。这是产品形态带来的天然优势，必须在设计阶段就固化进去。

## 三、头脑风暴议题逐条记录

### 议题 1：是否用 Image 生成漫画取代故事

**讨论过程**：
- 初始想法：用 Image-2 等图像生成 API，基于用户照片生成漫画风格图片来辅助记忆
- 进一步想法：用漫画 + Anki .apkg 双输出取代现有故事 + 填空题

**决定：放弃漫画取代故事，保留故事生成**

**放弃理由**：
1. 图像生成 API 调用延迟 3-8 秒，影响核心学习流程的即时性
2. 漫画风格的 prompt 工程需要大量调试才能出稳定效果，48h 内风险过高
3. 印尼语文字嵌入漫画格格里需要额外的排版处理
4. 漫画生成如果效果不稳定，demo 现场出奇怪的图会直接伤害产品印象
5. 现有的标注美图方案已经提供了足够强的视觉层，漫画不是必要升级

**保留的变体**：
- 可以考虑在 Anki 卡片上附一张 AI 生成的简洁插画（异步处理，不影响主流程）
- 但这是 Priority 4，在核心功能稳定后才考虑

### 议题 2：产品形态——拍照 → 标注图 + Anki

**结论**：这是正确的双输出形态，已在上文详述。

**补充讨论：两个输出的时序问题**

用户体验流程应该是：
1. 拍照（即时）
2. 生成标注美图（8-15 秒，可以有进度动画）
3. 展示美图，用户欣赏、保存、分享
4. 点击"学习模式"进入填空练习
5. 完成练习后，一键导出 Anki

"先美图，再学习"的顺序比"先学习，再看图"更符合用户心理——先给奖励，再要求付出。这也是为什么游戏先给你酷炫的开场动画，再让你过教学关卡。

**补充讨论：图片保存策略**

原图保存在文件系统 `images/` 文件夹，路径存 SQLite，不把 BLOB 塞进数据库（标准做法）。
标注图（叠加了印尼语文字的版本）另存一份，供用户下载分享。
Anki 卡片里附原图还是标注图？建议附**标注图**，因为复习时看到标注版本，对词汇的视觉锚定更强。

### 议题 3：痛点、用户、盈利、社交属性

#### 3.1 痛点重新定位

**旧痛点表述**：印尼语分级学习材料稀缺

**新痛点表述**（更具体，情感共鸣更强）：
> 你去旅行，看到一个好看的地方，想用当地语言描述它，现有工具全都帮不了你：Google 翻译只翻词，Duolingo 在教"大象喝水"，语言交换 app 要等真人。Lensa 的答案：拍一张照片，10 秒后你有一段地道的印尼语，可以学，可以分享，可以发给印尼朋友。

两个痛点层次：
- **表层痛点**：印尼语学习材料少、不够个性化
- **深层痛点**：没有一种自然的方式把真实生活经历转化为语言学习素材

Lensa 同时解决两个层次的痛点，这是核心差异化。

#### 3.2 用户画像（精确化）

**第一批用户（种子用户）**：
- 有 Anki 使用习惯
- 对东南亚有真实兴趣（旅行/工作/文化/恋爱）
- 在小红书 / Instagram 上活跃
- 对 AI 工具有好奇心，愿意尝试新产品

这个交集估计全球几十万人，国内主要集中在北上广深成都等城市的年轻白领和在校大学生。

**第二圈用户（口碑传播）**：
- 被第一批用户分享的美图吸引
- 未必有 Anki 习惯，但被内容形式打动
- 开始用 Lensa 创作自己的语言图片日记

第一圈→第二圈的传播链条，是真正的增长飞轮。内容即营销的逻辑在这里体现。

**我们自己就是用户**：
- 开赛前不会说印尼语
- 用 Lensa 学，48h 后能用印尼语描述身边场景
- 这个事实本身就是最好的 demo 证据和营销素材

#### 3.3 社交属性——正确打法

**错误打法**：多邻国式的 streak 打卡压力机制
- 焦虑大于乐趣
- 用户为了维持 streak 而不是为了学习而打开 app
- 和 Lensa"又能玩又能学"的定位冲突

**正确打法**：基于内容分享的社交
- 用户分享的不是"我今天学了 15 个词"，而是"我今天拍了一张照片，Lensa 把它变成了这个"
- 内容本身有美学价值，分享动机是真实的，不是焦虑驱动的
- 更接近小红书 / BeReal 的逻辑：分享一种生活方式，语言学习是这种生活方式的一部分

**可以加的社交功能（time-permitting）**：
- 分享按钮：直接导出带 Lensa 水印的标注图，一键保存到相册
- 生成分享文案：自动生成一段小红书风格的图片说明文字（中文+印尼语双语）
- 社区浏览：看其他用户拍的场景和对应的印尼语标注（这是真正的 UGC 内容社区）

#### 3.4 盈利路径（Hackathon 阶段不需要实现，Pitch 里可以埋一句话）

**短期（产品上线初期）**：
- Freemium：免费用户每天 3 张标注图，付费用户无限张
- 付费功能：高质量 TTS 发音 + 批量导出 Anki + 去除水印

**中期**：
- 订阅制：月费 / 年费，覆盖 API 成本 + 服务器
- 语言扩展：马来语、泰语、越南语、斯瓦希里语，每新增一个语言打开一个新市场

**长期（B2B）**：
- 卖给语言培训机构：为学员提供个性化内容生成工具
- 卖给企业：东南亚市场拓展团队的语言培训工具
- 客单价高得多，但需要产品成熟度和案例支撑

### 议题 5：产品会不会太玩具

**结论：产品形态本身不玩具，但叙事需要打磨**

**"玩具感"的两个来源**：

1. 功能链路太短，拍照之后没有持续的学习反馈
   - 解法：Anki 导出解决了这个问题，学习资产在 Lensa 之外继续存活

2. 技术深度没有被感知，用户觉得"这不就是 GPT 加个滤镜"
   - 解法：Pitch 里说清楚 i+1 约束、词表系统、FSRS 更新的工程深度
   - 在产品界面上适当展示词表状态（"你已经掌握了 47 个印尼语词汇"），让用户感知到系统在持续追踪

**"太简单"的认知偏差**：
真正被用户喜爱的产品往往比开发者预期的"简单"。Instagram 核心功能是拍照加滤镜分享，可以说很"玩具"，但它改变了内容分享的形态。产品是否玩具，不由技术复杂度决定，由用户获得的真实价值决定。

如果 Lensa 的标注图真的好看、真的能学到东西、真的能分享，它就不是玩具。

**真正需要担心的不是"太简单"，而是"体验不够流畅"**：
- 等待时间是否在用户可接受范围（< 15 秒）
- 标注图的视觉质量是否稳定（不能出奇怪布局）
- Anki 导入是否一次成功（需要 Wilson 真机测试验证）

## 四、技术改动设想（基于本次讨论）

### 改动 1：输出形态从"文字故事"升级为"标注美图"

**改动内容**：GLM 生成的内容从连续故事文本，改为针对图中多个物体的短标注文字（每个物体 2-3 句，口语风格，带情感色彩）

**改动范围**：
- `services/llm.py`：更新 prompt 模板
- `frontend/gradio_app.py`：增加图片叠字渲染逻辑（用 Pillow 在原图上叠加文字）

**新 Prompt 模板方向**：

```python
def build_annotation_prompt(objects, cefr, learned_words):
    return f"""
你是一个印尼语语言教学助手。
图片中识别到的物体：{objects}
用户水平：{cefr}
已掌握词汇：{learned_words[:60]}

请为图片中每个物体生成一段印尼语标注文字，风格要求：
- 口语化、有情感、像朋友在描述
- 每个物体 1-2 句话
- 90% 词汇来自已掌握词表，引入不超过 2 个新词
- 新词在括号里标注中文释义

输出严格 JSON，不输出其他内容：
{{
  "annotations": [
    {{
      "object": "物体名称（英文）",
      "text": "印尼语标注文字",
      "new_words": [{{"word": "新词", "translation_zh": "中文"}}]
    }}
  ],
  "caption": "整张图的一句话总结，诗意风格",
  "output_task": "填空题（把某个新词换成空格）"
}}
"""
```

**图片叠字渲染（Pillow）**：

```python
from PIL import Image, ImageDraw, ImageFont
import os

def render_annotation_image(image_path: str, annotations: list, caption: str) -> str:
    img = Image.open(image_path).convert("RGBA")
    draw = ImageDraw.Draw(img)
    
    # 尝试加载字体，fallback 到默认
    try:
        font = ImageFont.truetype("arial.ttf", 24)
        small_font = ImageFont.truetype("arial.ttf", 18)
    except:
        font = ImageFont.load_default()
        small_font = font
    
    # 简单分布策略：把标注均匀分布在图片四角和中心
    positions = [
        (50, 50), (img.width - 200, 50),
        (50, img.height // 2), (img.width - 200, img.height // 2),
        (50, img.height - 150), (img.width - 200, img.height - 150)
    ]
    
    for i, ann in enumerate(annotations[:len(positions)]):
        x, y = positions[i]
        text = f"— {ann['text']}"
        # 白色文字 + 黑色描边
        for dx, dy in [(-1,-1),(1,-1),(-1,1),(1,1)]:
            draw.text((x+dx, y+dy), text, font=font, fill=(0,0,0,200))
        draw.text((x, y), text, font=font, fill=(255,255,255,255))
    
    # 底部 caption
    draw.text((img.width//2, img.height - 60), caption,
              font=small_font, fill=(255,255,255,230),
              anchor="mm")
    
    output_path = image_path.replace(".jpg", "_annotated.jpg")
    img.convert("RGB").save(output_path, quality=92)
    return output_path
```

注意：这是最简版实现，视觉效果不如你们上传的那张图（那张图的排版和装饰更精细）。如果 Jianhao 有时间，可以用更复杂的排版算法（避免文字重叠、自动换行、添加装饰线条）。

### 改动 2：Anki 卡片附标注图而非原图

**理由**：复习时看到标注版本，视觉锚定效果更强，因为标注图本身包含了语境信息。

**改动范围**：`services/anki_builder.py` 中的图片字段，改为使用 `_annotated.jpg` 路径。

### 改动 3：添加分享按钮

**功能**：在 Gradio 界面上生成标注图后，加一个"保存图片"按钮，用户点击后下载标注图。

**实现**：Gradio 的 `gr.DownloadButton` 或者直接输出 `gr.Image` 组件，允许用户右键保存。

**可选**：自动生成一段小红书风格的配文（GLM 调用，让它生成一段中文 + 印尼语双语的图片说明文字）。

## 六、GLM 印尼语质量问题及解决方案

### 已知问题

GLM-4-Flash 的印尼语会出现以下错误：
- 词序错误（形容词后置规则：应该 `buku besar` 但有时写 `besar buku`）
- 介词混用（`di`静态位置、`ke`移动目标、`dari`来源，经常搞混）
- `me-` 前缀动词的构词规则错误

### 解决方案

**方案一（立即可用）**：在 prompt 里加语法约束段落

```python
GRAMMAR_GUARD = """
印尼语语法约束（严格遵守）：
1. 形容词后置：buku besar（大书），不是 besar buku
2. 介词方向：di（静态位置）、ke（移动目标）、dari（来源）
3. me- 前缀：及物动词加 me-，minum/makan 是例外不需要前缀  
4. 如果不确定某个词的语法，改用你确定正确的简单词
5. 口语风格可以省略主语，但语序必须正确
"""
```

**方案二（需要测试）**：换用 Qwen2.5 系列

阿里在东南亚有大量业务，Qwen 的印尼语训练语料比智谱多，语法错误率更低。建议在开赛前做一次对比测试：用同一个 prompt 分别跑 GLM-4-Flash 和 Qwen2.5-7B，人工判断哪个更自然。

**方案三（中长期）**：母语者 spot-check

比赛前找一个印尼语母语者快速检查 5-10 段样例，这是 Boyang 在 pitch 里承诺的"质量保证"，也是坦诚面对 limitations 的必要步骤。

## 七、TTS 音频方案修订

### 修订内容

从"MMS-TTS 本地推理"改为"Forvo API 批量下载 + MMS-TTS 作为 fallback"

### 理由

Forvo 是全球最大的单词发音众包数据库，印尼语有大量母语者录音，真人发音质量远比 TTS 合成音好。API 免费版每天 500 次请求，足够覆盖 A1/A2 词表的 500 个词。

### Wilson 的执行步骤

1. 注册 Forvo API（免费账号）
2. 写脚本批量请求词表里每个词的发音
3. 保存到 `audio/` 文件夹，文件名用词本身命名（`kopi.mp3`）
4. 没有 Forvo 音频的词，用 MMS-TTS 补全

### 降级方案

如果 Forvo API 注册麻烦或者请求失败率高，直接用 MMS-TTS。TTS 是 Priority 3，不是必做项，不要为了音频质量影响核心功能的稳定性。

## 八、冷启动定级测试设计

### 方案

新用户注册后，强制做一次 10 词快速定级测试，30 秒内完成，点"认识 / 不认识"。

### 测试词表（按 CEFR 分级）

**A1 测试词（4 个，日常最基础）**：
- `makan`（吃）
- `rumah`（家/房子）
- `besar`（大）
- `pergi`（去）

**A2 测试词（4 个，时态和条件）**：
- `sudah`（已经）
- `sedang`（正在）
- `kalau`（如果）
- `karena`（因为）

**B1 测试词（2 个，区分中高级）**：
- `meskipun`（尽管）
- `sehingga`（以至于/所以）

### 定级逻辑

| A1 认识数 | A2 认识数 | 初始 CEFR |
|---|---|---|
| < 3 | 任意 | A1 |
| ≥ 3 | < 2 | A1+（解锁 A1 全部 + 部分 A2）|
| ≥ 3 | ≥ 2 | A2 |

### 工程实现

这个功能在 Gradio 前端加一个启动页（Jianhao 负责），后端 `user_model.py` 里加一个初始化函数（Boyang 负责）。
