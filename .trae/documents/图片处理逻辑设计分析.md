# 前端后端图片处理逻辑设计分析

## 一、整体架构概览

本项目是一个印尼语学习应用，用户可以通过上传或拍摄图片来生成学习内容。系统采用前后端分离架构：

* **前端**: React + TypeScript

* **后端**: Python FastAPI

* **AI服务**: OpenAI GPT-4o + Anthropic Claude Sonnet

```typescript
const handleGenerate = useCallback(async (file: File) => {
  // 1. 开始生成状态
  dispatch({ type: 'GENERATE_START' });
  
  try {
    // 2. 调用API生成标注
    const result = await generateAnnotations(file, state.userId);
    dispatch({ type: 'GENERATE_SUCCESS', payload: result });

    dispatch({ type: 'RENDER_SUCCESS', payload: renderResult.imageUrl });
    
    // 4. 添加到画廊
    if (renderResult.imageUrl) {
      const galleryCard = { /* 构建卡片数据 */ };
      dispatch({ type: 'ADD_GALLERY_CARD', payload: galleryCard });
    }
  } catch (err) {
    // 错误处理
  }
}, [state.userId]);
```

### 3.3 AI标注服务

**文件位置**: [inputable/services/sonnet\_service.py](file:///d:/迅雷云盘/Flux_Lensa2026/inputable/services/sonnet_service.py)

#### 核心功能：

1. **CEFR分级提示词生成**：

   * A1级别：生成简单的"形容词+名词"标签（最多4个词）

   * A2级别：生成简单描述句（≤10个词）

   * B1级别：生成口语化、有情感的描述（2-3句）

2. **i+1词汇约束**：

   * A1: 最多引入1个新词

   * A2: 最多引入2个新词

   * B1: 最多引入3个新词

3. **降级策略**：

   * 尝试调用Claude Sonnet API

   * 失败后重试一次

   * 两次失败返回硬编码的默认结果

## 四、完整数据流

### 4.1 用户上传图片流程

```
用户操作
  ↓
ImageUploader组件
  ├─ 选择文件/拍照
  ├─ 创建预览
  └─ 触发onImageSelect(file)
  ↓
MyLearningPage组件
  └─ 传递给handleGenerate(file)
  ↓
useLensaApp Hook
  ├─ GENERATE_START (更新状态)
  ├─ 压缩图片为base64
  └─ 调用API: generateAnnotations(file, userId)
  ↓
前端API层
  ├─ 压缩图片 (max 1280px, quality 0.85)
  └─ POST /api/generate {user_id, image, mode}
  ↓
后端API路由
  ├─ 获取/创建用户
  ├─ 保存图片到磁盘
  ├─ 加载已学词汇
  └─ 调用AI服务
  ↓
AI标注服务
  ├─ 构建CEFR级别提示词
  ├─ 调用Claude Sonnet API
  ├─ 解析JSON响应
  └─ 应用i+1词汇约束
  ↓
后端返回
  ├─ 保存学习会话到数据库
  └─ 返回: {session_id, annotations, caption, output_task}
  ↓
前端处理响应
  ├─ GENERATE_SUCCESS
  ├─ 调用renderImage(sessionId)
  └─ RENDER_START
  ↓
渲染图片
  ├─ GET /api/render?session_id=xxx
  └─ 返回渲染后的图片URL
  ↓
更新UI
  ├─ RENDER_SUCCESS
  ├─ 显示标注后的图片
  ├─ 显示练习题
  └─ 添加到画廊
```

## 五、关键技术特点

### 5.1 图片压缩策略

* **前端**: Canvas压缩，最大1280px，JPEG质量0.85

* **后端**: Pillow压缩，最大1024px，JPEG质量85

* **目的**: 减少网络传输和API调用成本

### 5.2 状态管理

* 使用useReducer管理复杂状态

* 清晰的状态转换：upload → practice

* 支持错误处理和加载状态

### 5.3 AI集成

* 双AI架构：GPT-4o（图片识别）+ Claude Sonnet（教学标注）

* CEFR分级教学内容生成

* i+1词汇学习理论应用

### 5.4 数据持久化

* 用户信息存储

* 学习会话记录

* 词汇掌握情况跟踪

### 5.5 错误处理

* API调用失败重试机制

* 降级策略（返回默认内容）

* 用户友好的错误提示
