# 前端图片选择后点击发送再处理逻辑修复计划

## 问题分析

### 当前行为
1. 在 `ImageUploader.tsx` 中，当用户选择图片时：
   - 如果 `showGenerateButton` 为 `false`（默认值），会**立即**调用 `onImageSelect` 回调
   - 这导致图片选择后立即触发后端 API 调用

2. 在 `MyLearningPage.tsx` 中：
   - `ImageUploader` 组件没有传递 `showGenerateButton` 属性
   - `onImageSelect` 直接传递了 `handleGenerate` 函数

3. 结果：用户选择图片后立即发送请求到后端，没有机会预览或确认

### 期望行为
1. 用户选择图片后，显示图片预览
2. 用户确认后点击"发送"按钮
3. 点击发送后才调用后端 API 处理图片

## 解决方案

### 方案：修改 MyLearningPage 组件

修改 `MyLearningPage.tsx`，为 `ImageUploader` 组件添加：
- `showGenerateButton={true}` - 显示发送按钮
- `onGenerate={onImageSelect}` - 点击发送时调用后端

这样图片选择后只会显示预览，用户点击发送按钮才会触发后端处理。

### 步骤 3: 测试流程

3. 验证后端响应正常
